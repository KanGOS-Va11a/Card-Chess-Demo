# Day5-学习机制前置到主流程执行版

时间：2026-04-24

## 1. 文档定位

本文档用于承接本周开发计划中的 Day 5：

- Scene03 教学流程重排
- 学习机制前置到主流程
- C 菜单分页面首次进入教学弹窗
- 学习天赋与学习卡牌携带引导
- 教学文案整体重写优化

它不是长期设计文档，而是 Day 5 的直接执行清单。

## 2. Day 5 唯一目标

把“学习敌人技能”从当前项目里一个容易被忽略的成长点，提升为 `Scene03` 主流程中玩家必然经历、必然理解、必然实际操作一次的核心机制。

Day 5 完成的标准不是“学习牌已经存在”，而是满足：

1. 玩家会在 `Scene03` 中按自然认知顺序，完整经历一次学习机制教学
2. 玩家会被明确引导：
   - 完成第一场基础战斗
   - 理解 C 菜单的用途
   - 在 C 菜单里看到各页签的基础说明
   - 解锁学习相关天赋
   - 在构筑页携带学习卡牌
   - 进入第二场教学战并实际学习该敌人的技能牌
3. 第二个教学敌人是该敌人唯一一次学习机会
4. 所有教学文案都要简洁，但必须符合正常玩家的认知顺序

## 3. 你已确认的 Scene03 正式教学流程

Day 5 必须严格按下面顺序实现，不再偏移：

### 1. 第一场战斗：基础战斗教学

- 进入第一个敌人的战斗
- 战斗内弹窗教学基础机制
- 这一场只负责：
  - 基础动作
  - 基础卡牌
  - 战场元素
  - 回合资源认知

这一场不负责：

- 学习机制
- 学习天赋
- 学习卡牌携带

### 2. 战后回地图：C 菜单总览教学

- 打败第一个敌人回到地图
- 弹出 C 键菜单内容概括
- 引导玩家打开 C 菜单

这一步的职责是：

- 告诉玩家 C 菜单里有什么
- 告诉玩家成长、图鉴、构筑都在这里

但这一步仍然不把学习机制一次性讲完。

### 3. 打开 C 菜单后：分页面循序渐进教学

- 玩家打开 C 菜单后
- 各个页签会有简单弹窗教学
- 这些教学按玩家实际进入页签的顺序触发

核心目标：

- 让玩家先理解“这个页是做什么的”
- 再把玩家逐步引导到：
  - 学习相关天赋
  - 学习卡牌携带

关键约束：

- 这些页签教学只有在第一场基础战斗胜利后才会激活
- 在这之前，即使玩家打开 C 菜单，也不弹这些教学

### 4. 第二场战斗：学习机制与学习卡牌用法教学

- 进入第二个教学敌人的战斗
- 战斗内弹窗教学：
  - 学习机制是什么
  - 为什么这个敌人的技能值得学
  - 学习卡牌在战斗中怎么用

这一场是学习机制的正式抬前，不再只是菜单说明。

## 4. 第二个教学敌人的明确要求

这是本次流程中最关键的约束之一。

### 4.1 敌人类型

`Scene03` 里的第二个教学敌人：

- 敌人战斗内使用的敌人，必须与第一个基础教学战斗中的敌人相同

原因：

- 以后不会再出现这个敌人
- 所以这就是玩家学习该敌人技能卡的唯一机会

### 4.2 教学职责

第二个教学敌人不是普通重复战斗，它必须承担：

- 学习机制首次正式说明
- 学习目标来源说明
- 该敌人技能牌的唯一学习机会
- 学习卡牌用法说明

也就是说：

- 第一场负责“怎么战斗”
- 第二场负责“怎么学习敌人的能力，并把它变成自己的牌”

## 6. Day 5 的核心规则

### 规则 A：学习机制必须分两段抬前

第一段：

- 第一场战后
- 先让玩家知道 C 菜单是成长入口

第二段：

- 第二个教学敌人
- 正式把学习机制抬到战斗层

不要把这两段混成一个大杂烩弹窗。

### 规则 B：页签教学必须在正确阶段后才可触发

第一场基础战斗胜利前：

- 不触发任何学习相关页签教学

第一场基础战斗胜利后：

- 才允许 C 菜单页签教学进入候选状态

### 规则 C：页签教学必须是第一次进入某页时触发

不是：

- 打开 C 菜单就全弹
- 每次进某页都反复弹

而是：

- 达成流程前置后
- 第一次进入该页签时弹一次

### 规则 D：教学内容必须符合玩家认知顺序

推荐顺序必须是：

1. 先学“怎么战斗”
2. 再学“菜单里能做什么”
3. 再学“学习机制入口在哪”
4. 再学“学习牌怎么带进牌组”
5. 最后在第二场战斗里学“这张学习牌实际怎么用”

不能倒过来。

## 7. 推荐实现方案

## 7.1 Scene03 第一场战后：只做总览，不做深教学

保留 `Scene03PostBattleTutorialController`，但职责明确为：

- C 菜单总览
- 引导玩家打开 C 菜单

不要在这里一次性解释：

- 学习机制细则
- 学习天赋位置
- 学习卡牌构筑步骤

这些内容应该拆到后续页签教学和第二场战斗教学里。

## 7.2 新增 Scene03 页签教学控制器

建议新增一个专门监听 C 菜单页签的控制器，例如：

- `Scene03MenuPageTutorialController`
  或
- `SystemFeatureMenuTutorialController`

它的职责：

1. 监听 `SystemFeatureLabController` 当前激活页签
2. 检查流程是否已推进到第一场战后
3. 检查各页签首次教学 flag
4. 在正确页签第一次打开时弹 `PagedTutorialPopup`
5. 播放完成后记录 flag

### 它至少需要维护这些状态

- `scene03_phase_1_battle_cleared`
  - 第一场基础战斗完成
- `scene03_menu_page_tutorials_unlocked`
  - 第一场战后总览教学已播，页签教学可触发
- `tutorial_c_menu_status_first_open_shown`
- `tutorial_c_menu_talent_first_open_shown`
- `tutorial_c_menu_deck_first_open_shown`
- `tutorial_c_menu_codex_first_open_shown`
- `scene03_phase_2_learning_enemy_defeated`
  - 第二个教学敌人已完成

说明：

- 不是每个页签都必须讲很长
- 但“天赋页”和“构筑页”必须成为重点页签

## 7.3 C 菜单页签教学的职责划分

### 状态页

只讲：

- 这里能看角色当前状态
- 生命、能量、基础能力值和成长信息都能在这里确认

目标：

- 让玩家知道这里是信息总览，不做复杂成长操作

### 天赋页

这是 Day 5 的第一重点页签。

必须讲清：

1. 天赋不只是加数值
2. 有些节点会解锁新的玩法和新卡牌
3. 学习相关天赋节点在哪里
4. 玩家现在应该去点与学习相关的天赋

这里的引导应该直接把玩家指向：

- `talent_flex_learning`
  或你最终确定的学习入口节点

### 构筑页

这是 Day 5 的第二重点页签。

必须讲清：

1. 左边是可选牌池
2. 右边是当前携带牌组
3. 刚解锁的学习卡牌会出现在可选池里
4. 玩家需要把它加入当前构筑
5. 保存后才会在战斗中生效

### 图鉴页

只做轻量说明即可：

- 卡牌和敌人信息以后都能在这里查

Day 5 不需要把重点放在这里。

## 7.4 第二个教学敌人的战斗内教学

这是 Day 5 的第二重点实现。

战斗内弹窗教学必须讲清：

1. 你现在面对的是一个以后不会再出现的敌人
2. 这是你学习它技能牌的唯一机会
3. 学习机制会把敌人的能力转成你可使用的卡牌
4. 这类学习卡牌在战斗中的用途是什么

这里的重点是“敌人的能力会变成你的牌”，不是再重复讲基础战斗规则。

## 7.5 教学文案的整体优化原则

你已经明确要求：

- 文字尽量简洁
- 但必须讲清楚机制
- 要符合正常玩家认知顺序

所以 Day 5 文案重写必须遵守：

1. 一个弹窗只讲一个明确主题
2. 先讲“这是什么”
3. 再讲“我现在该做什么”
4. 不提前讲未来太远的系统
5. 不用设计者视角写法
6. 不用 debug 风格术语

### 需要被同步优化的已有文本

- 第一场基础战斗内教学文本
- 第一场战后 C 菜单总览文本
- 天赋页首次进入教学文本
- 构筑页首次进入教学文本
- 第二个教学敌人战斗内学习教学文本

收准 `Scene03` 的完整教学阶段状态。

目标：

- 把第一场基础战斗后、页签教学解锁、第二个教学敌人、第二场战斗后这些阶段 flag 定清楚

收工标准：

### Step 2

调整第一场基础战斗后的总览教学。

目标：

- 保留总览作用
- 不再把学习机制细则塞进去

收工标准：

- 第一场战后只负责引导玩家打开 C 菜单

### Step 3

做 C 菜单分页面首次进入教学触发。

目标：

- 达成前置后
- 第一次进入对应页签才弹一次

收工标准：

- 天赋页教学和构筑页教学都能独立只触发一次

### Step 4

把天赋页教学做成明确的学习节点引导。

目标：

- 玩家一进入天赋页就知道该点哪个学习相关节点

收工标准：

- 玩家能在引导下解锁学习相关天赋

### Step 5

把构筑页教学做成明确的学习牌携带引导。

目标：

- 玩家能理解并完成“加入学习牌 -> 保存构筑”

收工标准：

- 玩家不会只解锁学习牌却不知道要去构筑页携带

### Step 6

在 `Scene03` 落第二个教学敌人及其剧情/战斗内教学。

目标：

- 学习机制在主流程里有唯一一次明确战斗体验

收工标准：

- 第二个教学敌人承担学习机制正式教学职责
- 且该敌人与第一场基础敌人相同

### Step 7

统一重写所有涉及 Day 5 的教学文本。

目标：

- 文本简洁
- 顺序合理
- 玩家认知负担低

收工标准：

- 所有教学文本都符合“先理解当前任务，再理解机制”的顺序

## 9. Day 5 需要避免的错误

### 1. 不要在第一场基础战斗前就开放学习页签教学

原因：

- 会破坏认知顺序
- 玩家还没理由关心学习机制

### 2. 不要把学习机制继续藏在不显眼的天赋树角落

原因：

- Day 5 的目标就是把它抬到主流程前台

### 3. 不要把第二个教学敌人做成别的敌人

原因：

- 你已经确认这是该敌人唯一学习机会
- 这条体验必须和这个敌人的能力绑定

### 4. 不要让页签教学每次进入都重复弹

原因：

- 很快就会从引导变成打扰

### 5. 不要把文案写成设计说明书

原因：

- 玩家只需要当前步骤所需的信息
- 不是来读系统文档

## 10. Day 5 验收标准

只要以下条件成立，Day 5 就算完成：

1. 玩家在 `Scene03` 中完整经历：
   - 第一场基础战斗教学
   - 战后 C 菜单总览
   - C 菜单页签首次进入教学
   - 学习相关天赋解锁引导
   - 学习卡牌携带引导
   - 第二场学习机制战斗教学
2. 第一场基础战斗前打开 C 菜单，不会弹学习相关教学
3. 第一次进入天赋页会弹学习节点引导
4. 第一次进入构筑页会弹学习牌携带引导
5. 第二个教学敌人与第一场基础敌人相同，且是其唯一学习机会
6. 所有相关教学文案都比当前更简洁、更符合玩家认知顺序
