# 项目目录结构说明

## 1. 目标

本文件用于说明当前仓库的推荐目录边界，避免后续继续把地图、战斗、文档和临时原型混写到同一层。

## 2. 顶层目录

- `Assets`：贴图、字体、卡图等原始资源
- `Docs`：项目文档
- `Resources`：Godot 资源定义，当前以战斗资源为主
- `Scene`：地图与通用原型场景
- `Scene/Battle`：战斗专用场景
- `Scripts/Battle`：战斗域脚本
- `Scripts/Map`：地图域脚本
- `Template`：基础模板场景
- `devlog`：开发日志

## 3. Scripts 约定

### 3.1 地图域

地图域统一放在 `Scripts/Map`，当前建议继续按以下分层维护：

- `Scripts/Map/Actors`：地图可控角色，例如 `Player`
- `Scripts/Map/Interaction`：地图交互对象与交互基类
- `Scripts/Map/Controllers`：地图场景级控制器
- `Scripts/Map/Transitions`：地图与战斗的公共切换助手

### 3.2 战斗域

战斗域继续放在 `Scripts/Battle`，按现有模块拆分：

- `Actions`
- `AI`
- `Arakawa`
- `Board`
- `Boundary`
- `Cards`
- `Data`
- `Encounters`
- `Presentation`
- `Rooms`
- `Shared`
- `State`
- `Turn`
- `UI`
- `Visual`

## 4. 维护规则

- 地图恢复逻辑只放在 `MapSceneController`
- 进入战斗的公共逻辑只放在 `MapBattleTransitionHelper`
- 地图交互对象优先继承 `InteractableTemplate`
- 新增文档放在 `Docs`
- 从本次整理开始，新增开发记录统一以中文写入 `devlog`
