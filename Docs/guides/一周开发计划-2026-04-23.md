# 一周开发计划

时间范围：2026-04-23 至 2026-04-29

## 1. 本周唯一完工目标

本周只追求一个结果：

**完成“星港外沿 -> 后巷入口 -> 搭船客基地 -> 区域Boss”这一条可交付切片的底层重构与主流程打通。**

这条切片必须满足：

- 地图交互以格子为准
- 对话与剧情弹窗统一
- 前置 UI 出现时锁死移动和交互
- 战斗返回地图不刷新地图物品与一次性剧情
- 构筑结果能稳定进入战斗
- 学习机制在流程中被明显前置和展示

本周不追求：

- 全量美术替换完毕
- 全量音乐替换完毕
- 全量非线性支线铺开
- 所有 AI 彻底重写

## 2. 本周交付边界

### 2.1 做完算完成

1. 地图交互底层从球形碰撞切到格子交互
2. 统一剧情 / 教程 / 普通对话弹窗表现
3. 统一输入锁与动作锁
4. 地图返回状态保留稳定
5. 构筑真源重做完成
6. 学习机制在主流程中被明确抬到高优先级位置

### 2.2 做不完也不能拖主线

1. 天赋界面深度美术重构
2. 全区域背景与音乐全量替换
3. 全敌人 AI 完全重做
4. 提案全文 polish

## 3. 每天的工作安排

## Day 1

主题：地图交互底层改造方案定稿并开始落代码

- 以格子为准重新定义地图交互判定
- 明确交互对象、剧情触发区、出生点、敌人刷新点应使用的图层
- 清理仍依赖球形碰撞交互的主链节点
- 确定剧情触发与普通交互的统一入口

收工标准：

- 格子交互的代码入口明确
- 主链场景中不再依赖球形碰撞作为核心交互逻辑

## Day 2

主题：统一对话与剧情文本系统

- 统一剧情文本弹窗、教程弹窗、普通对话弹窗的调用方式
- 确定唯一文本表现链路
- 对话弹出时锁死角色输入
- 对话关闭时恢复正确状态，不再残留行走动画

收工标准：

- 主链对话不再混用多套弹窗
- 对话期间角色不能移动、不能乱开菜单、不会卡动作

## Day 3

主题：地图状态保留与返回逻辑修正

- 修复战斗返回地图后物品刷新问题
- 增补必要的进度标记变量
- 把一次性箱子、剧情触发区、敌人清除状态统一纳入快照/状态链路
- 收口地图返回后的场景状态恢复

收工标准：

- 战斗返回地图不再重置关键交互物
- 一次性内容能够稳定保持已完成状态

## Day 4

主题：构筑系统真源重做

- 重新实现构筑数据真源
- 修复左侧候选区、右侧携带区、影响因子、战斗初始化不同步
- 保证“保存后生效”的规则明确稳定
- 清掉靠别的操作才刷新的临时方案

收工标准：

- 构筑界面显示稳定
- 战斗内初始化与构筑结果一致

## Day 5

主题：学习机制前置到主流程

- 调整教程和区域流程，让学习机制不再只是天赋角落中的附属内容
- 在流程中至少安排一次明确的“学习敌人能力 -> 回到构筑确认 -> 后续使用”的体验
- 同步调整图鉴 / 构筑 / 奖励显示

收工标准：

- 学习机制在 30 分钟内能被玩家明确感知
- 提案中可据此描述玩家情绪时刻

## Day 6

主题：敌人 AI 与战场题面优化

- 优化敌人绕障与寻路优先级
- 让敌人意图更明确，不轻易被障碍卡住
- 优化房间题面，让学习机制、战场改造和不同敌人职责更容易被看懂
- 同时检查是否已有非线性流程雏形

收工标准：

- 敌人行为不再显著破坏体验认知
- 关键战斗的题面差异能支撑区域切片

## Day 7

主题：本周收口与提案同步

- 修复本周改动引入的文本溢出、错字和明显表现问题
- 同步更新主任务文档与提案草案中的切片描述
- 归纳本周完成内容，明确下周继续项
- 不新增新 md，只更新现有活跃文档

收工标准：

- 一条完整切片进入“可由开发者本人审查运行时内容”的状态
- 提案描述与游戏现状不脱节

## 4. 本周执行约束

1. 不新增第二份主任务拆分文档
2. 只有代码功能变更才写 `devlog`
3. 纯文档整理、归档、提案润色不写 `devlog`
4. 不由助手执行 Godot 运行测试
5. 每天结束时，只允许更新：
   - 本文件
   - 主任务文档
   - 提案草案

## 5. 本周结束的验收标准

本周是否成功，只看以下问题是否成立：

- 玩家是否已经能在一个完整区域切片中体验到学习机制
- 主流程是否已摆脱球形碰撞交互与混乱弹窗方案
- 构筑结果是否可靠进入战斗
- 战斗返回地图是否不再刷新关键交互物
- 提案书是否已经能据此描述一个明确、完整、可验证的核心切片
