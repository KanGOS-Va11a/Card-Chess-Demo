# 美术需求文稿（初稿）

## 1. 文档定位

本文档用于基于当前项目已实现内容与现有世界观设定，整理一份面向概念美术、角色美术、场景美术、UI 美术与特效美术的初步需求文稿。

这不是最终定稿，而是一份能指导前期视觉探索、风格统一和资源排期的工作文档。

适用对象：

- 主美 / 概念设计
- 角色原画 / 立绘 / 像素角色
- 场景与关卡美术
- UI / HUD 设计
- 特效 / 动画 / 视觉包装
- 外包沟通与需求对齐

## 2. 项目一句话视觉定义

这是一部发生在高压监管宇宙中的“星际牛仔 + 故障 AI + 铁幕科幻战棋”的作品，整体气质应当同时具备：

- 边境漂流感
- 高科技压迫感
- 废墟中的神秘奇迹感
- 个人英雄式的浪漫与孤独

不是纯赛博朋克，也不是纯西部片，而是“被宇宙秩序抛弃的牛仔，在工业废墟与算法铁幕之间生存”的视觉混合体。

## 3. 世界观视觉核心

## 3.1 三个最重要的视觉支柱

### 1. 星际牛仔感

关键词：

- 旧大衣
- 红色方巾
- 金属皮靴
- 旧时代旅者气质
- 孤独、硬朗、冷感

这部分决定角色为什么不是一个普通科幻士兵，而是一个“有传奇感的边境漂流者”。

### 2. 铁幕时代感

关键词：

- 绝对秩序
- 网络监控
- 透明牢笼
- 冷色光源
- 统一制式视觉
- 无处不在的识别系统与监管符号

这部分更多会体现在：

- 星港
- 联盟设施
- 公共系统 UI
- 警报、扫描、验证、身份识别装置

### 3. 荒川与“非自然奇迹”

关键词：

- 幽蓝荧光
- 蓝白高压电弧
- 故障艺术
- 像素崩裂
- 半透明投影
- 旧工业机械与觉醒 AI 的混合感

这部分是项目最强的差异化视觉来源，必须形成一眼可识别的视觉语言。

## 3.2 主色调建议

建议不是一套平均化的“赛博紫蓝霓虹”，而是分层使用：

- 主基底：
  - 冷灰
  - 铁锈黑
  - 舰船工业绿灰
  - 脏白
- 主角识别色：
  - 暗红 / 血红方巾
- 荒川识别色：
  - 幽蓝
  - 蓝白电弧
  - 故障彩色像素块
- 危险 / 联盟秩序色：
  - 冷白
  - 警告橙
  - 高压红

## 4. 当前项目内容对应的美术需求范围

根据现有代码与文档，当前最需要支撑的玩法内容包括：

- 地图探索原型
- 战斗棋盘与房间池
- 主角与敌人单位
- 障碍物与地形差异
- 卡牌与构筑 UI
- 荒川能力与战斗特效
- 牌堆查看与牌组构筑入口

这意味着美术工作不能只看角色，还要同时考虑：

- 战场阅读性
- 信息层级
- 卡牌识别性
- 房间地形池的变化感

## 5. 角色美术需求

## 5.1 主角：星际牛仔

### 角色定位

主角是一个“失忆的、拥有不该存在之身躯的人类 / 非人类边界体”。

视觉上必须同时传达：

- 人类外观
- 不完整的过去
- 可疑的身体构成
- 静默危险感

### 核心识别元素

- 落满星尘的旧大衣
- 红色方巾
- 神秘手提箱
- 身上存在少量非自然植入痕迹
- 并非整齐军用装备，而是混合了拾荒、流浪、工业与实验痕迹

### 气质要求

- 不要做成嘻哈或夸张潮流感牛仔
- 不要做成常规机甲士兵
- 应更接近：
  - 边境亡命徒
  - 冷硬漂流者
  - 失去过去但战斗本能极强的人

### 需要的资源类型

- 角色立绘 / 关键视觉图
- 战斗像素角色 / 棋盘单位
- 头像 / 状态栏图标
- 受伤 / 防御 / 逃跑 / 待机等动作表现

## 5.2 荒川：AI 队友

### 角色定位

荒川不是传统“可爱导航 AI”，而是一个：

- 带有旧时代工业出身
- 已经超越工具身份
- 轻快但带审判味道
- 玩弄秩序于股掌之间的流窜 AI

### 核心识别元素

- 半透明投影少女
- 故障像素重构出场
- 蓝白主色
- 机械工业来源的细节痕迹
- 可以有一点旧日日系工业品牌残留感

### 视觉风格要求

- 不能是普通二次元陪伴精灵
- 也不能是完全冰冷的纯全息 UI
- 她需要“既像人，又始终不像人”

### 需要的资源类型

- 角色概念图
- 对话头像 / 半身像
- 荒川轮盘 UI 图标
- 能力发动特效
- 投影出场 / 故障崩裂效果

## 5.3 搭船客：早期敌对势力

### 敌人定位

他们是边境掠夺者，不是正规军，也不是怪物。

应体现：

- 脏
- 狠
- 杂牌军
- 临时拼装武器
- 擅长靠数量与突袭压制

### 视觉方向

- 防具与服饰不统一
- 武器带明显改造痕迹
- 比主角更粗粝、更廉价、更边缘

### 需要的资源类型

- 基础敌人 2 到 4 套轮廓差异
- 近战 / 远程 / 精英 的层级变化
- 头像 / 棋盘单位 / 倒地或受击反馈

## 6. 场景与环境美术需求

## 6.1 第一章核心场景：报废边缘的幽灵船

### 场景定位

这是玩家最早接触世界的地方，必须让人一眼理解：

- 这里曾经高度工业化
- 现在正快速走向毁灭
- 这不是普通事故，而是暴力失控后的残骸

### 场景关键词

- 扭曲舱壁
- 断裂灯带
- 失压警报
- 金属疲劳
- 泄露蒸汽
- 被破坏的舱门
- 残留血迹与打斗痕迹

### 视觉目标

玩家一进入就要感觉：

- 紧迫
- 冷
- 脏
- 危险
- 没人会来救你

## 6.2 边境星港

### 场景定位

这是秩序与边缘的交汇地。

既要有：

- 星际联盟的冷酷秩序

又要有：

- 边境世界的灰色地带

### 关键词

- 巨大的身份验证装置
- 联盟制式灯牌与标语
- 廉价维修摊位
- 走私感
- 港口钢架、停泊塔、货箱
- 数据屏与旧工业结构并存

## 6.3 房间池 / 地形池

这是当前项目一个非常值得美术重点参与的部分。

### 设计目标

通过丰富的房间地形池设计，让每一场遇敌战斗都带有 roguelike 随机感。

这种随机感不是“敌人数值变一变”，而是：

- 战场空间关系发生变化
- 障碍物、通道、开阔区、掩体、缓速区的组合发生变化
- 玩家熟悉的敌人组合，在不同地形上变成完全不同的局

### 美术上要解决的问题

- 房间变化要明显，但不混乱
- 地形差异要一眼看懂
- 棋盘阅读性不能被装饰性吞掉

### 当前推荐的房间池方向

- 狭长通道型
  - 适合压迫与远程压制
- 开阔交错型
  - 适合走位和风筝
- 掩体密集型
  - 适合绕后与视线博弈
- 破碎废墟型
  - 适合高风险近战推进

## 7. 战斗视觉需求

## 7.1 棋盘阅读性

当前战斗是小棋盘高密度信息结构，美术必须优先保障：

- 格子清晰
- 单位一眼分阵营
- 可破坏 / 不可破坏 / 缓速地形能一眼区分
- 特效不遮挡决策

这类项目的视觉优先级，应是“先看清，再好看”。

## 7.2 单位与地形层级

建议层级关系：

- 地板
- 地形标记
- 障碍物
- 单位
- 高亮与路径预览
- 数值与伤害飘字
- 特殊能力特效

不要让背景或装饰把单位轮廓吃掉。

## 7.3 动画与移动表现

当前项目已经把移动改成逐格或慢速移动，这意味着后续动画资源可以围绕：

- 每格位移
- 起步 / 收步
- 攻击前倾
- 受击后仰
- 防御站姿
- 荒川能力触发

来设计，而不是做大跨度瞬移动画。

## 8. UI 与卡牌美术需求

## 8.1 HUD 总体气质

UI 不应走“高饱和霓虹 + 炫酷玻璃面板”的常见赛博套路。

更适合的方向是：

- 小型战术终端
- 粗粝工业面板
- 带一点旧时代电子设备感
- 干净但不华丽

玩家打开 UI 时应感觉是在用一个老旧但可靠的战术设备。

## 8.2 卡牌视觉

卡牌是这个项目的重要识别面。

### 当前需求

- 攻击卡、技能卡、治疗卡在颜色和符号上要易区分
- 即使没有插画，也要靠构图和图标读懂卡牌用途
- 插画加入后，要强化卡牌身份，而不是喧宾夺主

### 当前卡牌图需求建议

第一阶段不必给每张卡做高成本完整插画，可以采用：

- 统一风格图标 / 符号插画
- 局部色块 + 类型图形
- 少数核心卡做重点插画

这样更适合原型期扩卡。

### 牌堆查看

当前项目已经支持在查看抽牌堆、弃牌堆和消耗堆时显示卡面，而不只是卡牌名。

这意味着卡牌视觉组件要兼容：

- 手牌区
- 牌堆弹窗
- 后续构筑界面

所以卡牌美术必须统一，不要做三套完全不同的卡面体系。

## 8.3 局外构筑界面

局外构筑界面当前是最小原型入口，后续正式版本需要解决：

- 可选牌库的分组与筛选
- 已选牌组的可读性
- 点数预算的即时反馈
- 非法构筑的提示清晰度
- 天赋分支与卡池的关联可视化

## 9. 特效与 VFX 需求

## 9.1 荒川能力特效

这是当前最应该优先强化的一类特效。

关键词：

- 幽蓝脉冲
- 蓝白高压电弧
- 像素故障
- 数字污染
- 非自然重构

荒川的特效必须跟普通攻击、普通技能明显不是一个系统。

## 9.2 战斗反馈特效

至少需要在后续逐步强化：

- 普通攻击命中
- 护盾获得
- 治疗回复
- 防御姿态
- 逃跑成功
- 牌堆查看 / 卡牌强化反馈

## 10. 优先级建议

为了更适合当前项目阶段，建议美术优先级如下：

### 第一优先级：决定项目识别度

- 主角概念图
- 荒川概念图
- 幽灵船 / 边境星港 气质图
- 战斗棋盘地形语言
- 卡牌统一视觉语言

### 第二优先级：支撑当前可玩内容

- 主角战斗单位
- 搭船客基础敌人
- 三类障碍物
- 卡牌插画 / 图标
- HUD 优化

### 第三优先级：提升沉浸感

- 荒川特效
- 投影与故障表现
- 房间池视觉变化
- 星港和联盟设施的风格扩展

## 11. 当前风险与注意事项

## 11.1 风格跑偏风险

最需要避免的是：

- 做成普通赛博朋克
- 做成普通太空海盗
- 做成普通二次元 AI 伙伴

本项目真正有价值的地方，在于这些元素的混合比例非常特殊。

## 11.2 信息可读性风险

战斗棋盘和卡牌系统都高度依赖阅读速度，所以必须警惕：

- 地形过花
- 特效过满
- 卡牌信息堆积失控

## 11.3 房间池价值被低估的风险

房间池不是简单换背景，而是项目 roguelike 感的重要来源。

所以环境美术在这个项目里不是陪衬，而是玩法的一部分。

## 12. 结论

这份美术需求文稿的核心不是要求“做很多图”，而是帮助项目先建立统一的视觉语言。

对这个项目来说，美术最重要的任务不是装饰，而是：

- 让星际牛仔身份成立
- 让荒川的异常感成立
- 让铁幕时代的压迫感成立
- 让房间池与地形池真正成为玩法的一部分
- 让卡牌与棋盘在视觉上形成统一系统

只要这几条抓住了，这个项目的美术就不仅是好看，而是会真正参与塑造玩法体验。
