# 项目总体需求表

## 1. 文档定位

本文件只记录当前仓库对应的真实完成度。

状态含义：

- `已实现`：当前代码里已有可运行实现
- `部分实现`：已有主链路，但还不是完整系统
- `未实现`：当前仓库没有对应能力

## 2. 顶层状态总表

| ID | 模块 | 当前状态 | 说明 |
| --- | --- | --- | --- |
| T-01 | 主入口 | 已实现 | 当前主入口是地图场景 `Scene/Mainlevel.tscn` |
| T-02 | 全局共享状态 | 部分实现 | `GlobalGameSession` 已保存玩家快照、荒川能量和切换边界数据，但不是完整存档容器 |
| T-03 | 地图交互原型 | 已实现 | 已有 `Player`、`Npc`、`Chest`、`HealStation`、`SceneDoor`、`BattleEncounterEnemy` |
| T-04 | 战斗原型 | 已实现 | 已有 board、房间、视图、HUD、回合、卡牌、荒川、敌方最小 AI |
| T-05 | 地图进入战斗闭环 | 已实现 | 地图可写入 `BattleRequest` / `MapResumeContext` / `EncounterId` 并进入战斗 |
| T-06 | 战斗失败返回地图 | 已实现 | 战斗失败后可写回 `BattleResult` 并恢复地图位置 |
| T-07 | 完整胜利结算 | 未实现 | 当前没有正式掉落、奖励和胜利回写规则 |
| T-08 | 存档系统 | 未实现 | 当前没有正式存档读写结构 |
