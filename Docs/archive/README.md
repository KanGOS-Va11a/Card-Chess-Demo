# 文档归档说明

## 1. 归档原则

`Docs/archive` 用于存放以下内容：

- 已完成阶段的冲刺文档
- 旧提案支撑材料
- merge / 交接过程记录
- 不再作为当前开发主入口的阶段性清单

归档的目的不是删除历史，而是把当前有效文档数量压到可维护范围。

## 2. 当前归档结构

- `初赛阶段/冲刺计划`
- `初赛阶段/提案支撑`
- `初赛阶段/合并与交接`
- `技术归档`

## 3. 使用规则

- 归档文档默认只读，不继续追加
- 若当前开发确实需要复用某份旧文档，优先提炼出结论写回活跃文档，而不是重新激活旧文档
- 归档目录中的历史链接不保证持续维护
