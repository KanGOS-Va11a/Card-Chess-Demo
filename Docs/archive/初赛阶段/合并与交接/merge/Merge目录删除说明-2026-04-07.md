时间：2026-04-07

## 1. 当前结论

`Merge` 目录现在可以删除。

原因：
- 主项目运行与编译已不依赖 `Merge`
- donor 地图主链路已并入主项目
- 旧 donor 参考材料已迁移到 `Docs/merge`

## 2. 删除前已完成的文档迁移

- `Docs/merge/地图层项目合并方案-2026-04-06.md`

- `Docs/merge/Merge目录删除说明-2026-04-07.md`

## 3. 删除 Merge 后的影响

### 3.1 不会影响的内容

- 主项目编译
- 主项目运行
- donor 地图主链路
- 当前 battle / map 主链路

### 3.2 会失效的内容

- 历史 `devlog` 中对 `Merge/...` 的原始路径提及
- 原 donor 目录作为对照仓的直接查看能力

这类失效只影响历史回溯，不影响当前项目运行。

## 4. 删除后的额外建议

删除 `Merge` 后，可选再做一项清理：
- 检查 [newproject.csproj](D:/Godot/newproject/Card-Chess-Demo/newproject.csproj) 中的
  - `Compile Remove="Merge\\**\\*.cs"`

这条保留也不影响项目。  
如果想进一步收干净，可以之后再移除。

## 5. 推荐做法

删除步骤建议：
1. 先关闭 Godot 编辑器
2. 删除 `Merge` 目录
3. 重新打开主项目
4. 用 donor 的 `Scene01` 再验证一次主链路

如果删除后仍看到旧依赖弹窗，优先怀疑编辑器缓存或旧标签页恢复状态，而不是当前主项目又重新依赖了 `Merge`
