## 1. Day1 实际落点

目前可以认为 Day1 已完成的部分：

- 地图主入口、战斗进入、战斗返回主链路已稳定可跑。
- 旧 root scene、旧 MainPlayer、旧测试场景的大部分混线已清理归档。
- `C` 菜单主框架仍在，状态 / 背包 / 天赋 / 图鉴 / 构筑页还能工作。
- 战斗中一批关键卡牌、地形、击退、学习、结算链路已有实现基础。
- 战斗镜头移动与地图侧系统菜单仍在持续收口，代码可编译。

Day1 未完全收尾、已顺延到 Day2 的问题：

- 天赋页视觉与交互仍未正式收口，尤其是背景、布局与可视范围。
- 菜单里仍有残留乱码文本。
- 角色能力树与卡牌树虽然已能看见，但布局仍需要更精细的策划化整理。
- 学习 / 战后奖励 / 成长回写仍缺少一版正式验收。

结论：

Day2 不再把重点放在继续大规模修旧链路，而是要把“战斗完成后玩家得到什么、菜单能否看到成长结果、内容如何继续推进”做成闭环。

## 2. Day2 当日目标

Day2 的目标不是继续补一堆零散内容，而是把下面这条链打实：

`地图遇敌 -> 战斗 -> 学习 / 奖励 / 经验 -> 返回地图 -> C 菜单确认成长结果`

当天结束时，至少要满足：

- 玩家能通过 1 场普通战和 1 场带学习目标的战斗，稳定获得成长结果。
- 战后奖励、经验、已解锁卡牌、构筑状态、菜单显示彼此一致。
- Day1 遗留的天赋页至少达到“可展示、可浏览、可读懂”的展示底线。

## 3. Day2 必做内容

### 3.1 学习机制闭环

目标：
- `学习` 卡不只是能记录，而是能通过战斗流程稳定产出对应学习结果。

必做项：
- 明确普通学习牌与特色学习牌的判定条件。
- 敌人“可学习状态值”的置真、重置、结算时机固定下来。
- 至少保证 1 个普通敌人和 1 个带特色行为的敌人，能稳定产出不同学习结果。
- 学习成功后，`GlobalGameSession` 的已解锁卡牌状态要立即可见。

验收：
- 战斗内成功使用 `学习` 后，战后结算能明确看到学到的卡牌。
- 返回地图打开 `C` 菜单时，图鉴和构筑能看到对应变化。

### 3.2 战后结算正式化

目标：
- 把“战斗结束时发生什么”从临时逻辑变成一套稳定入口。

必做项：
- 统一胜利 / 失败 / 逃跑三种结算路径。
- 战斗只掉经验值，升级提供专精点，这一套数值回写要稳定。
- 战后奖励面板至少明确显示：
  - 经验值
  - 学到的卡牌
  - 物品 / 装备占位奖励（如当前流程里已经接好）
- 地图上的敌人节点与剧情标记要在胜利后正确消失。

验收：
- 打完敌人后不会再次回到地图重复触发同一敌人 / 剧情。
- 同一场战斗不会重复结算奖励。

### 3.3 菜单与成长结果对齐

目标：
- 战后回到地图后，玩家能在 `C` 菜单里“看懂自己刚刚获得了什么变化”。

必做项：
- 修正经验值、下一级经验、专精点显示不一致的问题。
- 已解锁卡牌在图鉴、构筑、学习结果之间的数据一致。
- 装备、天赋、构筑对战斗参数的影响在菜单显示中尽量反映。
- 清理当前菜单里最影响阅读的乱码字段。

验收：
- 战后结算看到的结果，返回地图后在菜单里能逐项对上。

### 3.4 天赋页收口到“展示可用”

目标：
- Day2 不再把天赋页当成纯技术试验，而是做到 demo 可展示。

必做项：
- 角色能力树按前置层级整理好。
- 卡牌树三分支视觉上继续保持明显分离。
- `WASD` 浏览、默认视角、节点可达范围全部稳定。
- 背景图显示问题彻底定位并修复。
- 至少把常驻提示、专精点显示、节点标题做成无歧义状态。

验收：
- 不需要解释也能看懂“左边是三大卡牌分支，右边是角色能力树”。
- 录屏时不会再出现一打开就在错误位置或背景像没加载的情况。

## 6. 当天执行顺序

建议严格按这个顺序推进：

1. 先修战后结算入口与地图回写稳定性。
2. 再把学习机制做成可验收闭环。
3. 接着修菜单里的成长结果显示一致性。
4. 最后收口天赋页展示与浏览体验。

顺序原因：
- 如果先沉进去继续修天赋页，很容易再次把一整天耗在 UI 上，但 Day2 的核心价值其实是成长闭环。

## 7. 当天验收用例

### 用例 A：普通胜利

- 地图进入普通敌人战斗。
- 战斗胜利。
- 看到经验值结算。
- 返回地图后敌人消失。
- 打开 `C` 菜单，确认经验、专精点、构筑状态可读。

### 用例 B：学习胜利

- 带 `学习` 卡进入学习目标战斗。
- 敌人进入可学习状态。
- 成功学到普通或特色卡牌。
- 战后结算中显示学到的牌。
- 返回地图后在图鉴 / 构筑里确认卡牌已解锁。

### 用例 C：失败或逃跑

- 触发失败或逃跑。
- 不发放错误的胜利奖励。
- 返回地图后 session 状态不污染。

## 8. Day2 收工标准

Day2 只有在下面条件同时满足时才算完成：

- 1 场普通战与 1 场学习战都能完整跑通。
- 战后奖励与菜单显示一致。
- 学习机制至少对 2 类结果生效。
- 地图侧敌人清除与剧情状态不再重复触发。
- 天赋页不再阻碍展示录屏。
