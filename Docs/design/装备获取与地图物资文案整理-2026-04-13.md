# 装备获取与地图物资文案整理

## 1. 当前链路整理

### 1.1 装备获取链路

当前装备并不是通过单独的“装备掉落系统”进入角色面板，而是统一先进入背包物品字典，再由装备系统识别其 `item_id` 是否为装备。

实际链路如下：

1. 地图交互物体给出物品
   - 入口脚本：
	 - `res://Scripts/Map/Interaction/Chest.cs`
	 - `res://Scripts/Map/Interaction/Cabinet.cs`
   - 配置入口：
	 - `res://Scripts/Map/Interaction/InteractableContainerConfig.cs`
   - 每个交互物体可配置：
	 - `InteractionTexts`
	 - `GrantedItems`

2. 交互物体把物品写入背包
   - 使用 `InventoryDelta`
   - 最终调用：
	 - `GlobalGameSession.ApplyInventoryDelta(...)`
	 - 文件：`res://Scripts/Battle/Shared/GlobalGameSession.cs`

3. 背包界面识别装备
   - `GlobalGameSession.FindEquipmentDefinition(itemId)`
   - 若 `item_id` 存在于 `EquipmentCatalog`，则视为装备
   - 相关文件：
	 - `res://Scripts/Battle/Equipment/EquipmentCatalog.cs`
	 - `res://Scripts/Battle/Equipment/EquipmentDefinition.cs`

4. 玩家在 C 菜单装备页执行装备
   - `GlobalGameSession.TryEquipItem(slotId, itemId, out failureReason)`
   - 装备后的属性由 `PlayerStatResolver` 统一结算

### 1.2 地图物资获取链路

当前地图物资主要来自三类来源：

1. 箱子
   - 场景：
	 - `res://Scene/InteractableItem/Chest.tscn`
	 - `res://Scene/InteractableItem/Chest01.tscn`
	 - `res://Scene/InteractableItem/Chest03.tscn`
   - 逻辑：
	 - `res://Scripts/Map/Interaction/Chest.cs`

2. 柜子
   - 场景：
	 - `res://Scene/InteractableItem/Cabinet01.tscn`
	 - `res://Scene/InteractableItem/Cabinet02.tscn`
   - 逻辑：
	 - `res://Scripts/Map/Interaction/Cabinet.cs`

3. 回复站
   - 场景：
	 - `res://Scene/InteractableItem/HealingStation.tscn`
   - 逻辑：
	 - `res://Scripts/Map/Interaction/HealStation.cs`
   - 功能：
	 - 回满 HP
	 - 回满荒川能量
	 - 播放 `heal` 音效

## 2. 当前文本实际来源

### 2.1 装备文本来源

装备名称与描述当前写在：

- `res://Scripts/Battle/Equipment/EquipmentCatalog.cs`

即 `new EquipmentDefinition(itemId, displayName, slotId, description, ...)`

### 2.2 背包通用物资文本来源

普通材料/消耗品名称与描述当前写在：

- `res://Scripts/Map/UI/SystemFeatureLabController.cs`

主要入口：

- `GetInventoryItemDisplayName(string itemId)`
- `GetInventoryItemDescription(string itemId)`

### 2.3 地图交互文本来源

每个交互体当前可写：

- `PromptText`
- `InteractionTexts`

说明：

- `PromptText`：交互提示，例如“打开宝箱”“搜索柜子”
- `InteractionTexts`：一个字符串数组，按交互次数依次显示
- 当交互次数超过数组长度后，停留在最后一条

## 5. 当前地图交互体建议补文案项

### 5.1 箱子类

建议为每种箱子预留以下文本：

- `PromptText`
- `InteractionTexts[0]`
- `InteractionTexts[1]`

推荐写法：

- `PromptText`：打开宝箱 / 检查箱子
- 首次交互文本：突出内容物、环境感、玩家判断
- 第二次及之后：箱子空了 / 只有灰尘 / 已经没有东西

### 5.2 柜子类

建议为每种柜子预留以下文本：

- `PromptText`
- `InteractionTexts[0]`
- `InteractionTexts[1]`

推荐写法：

- `PromptText`：搜索柜子 / 翻找储物柜
- 首次交互文本：偏“搜刮”“翻找”语气
- 后续文本：强调已被搜空

### 5.3 回复站

当前已有系统文本：

- `HealText`
- `AlreadyFullText`

建议补文案方向：

- 恢复时更有“站点/装置感”
- 满状态时更简短，避免反复交互啰嗦

## 6. 推荐你优先补的文案

如果按 demo 优先级，建议先写这三组：

1. 所有正式装备的描述
   - 因为这些会直接进入角色构筑和装备界面

2. 五个基础地图物资的名称与描述
   - 因为背包、箱子、柜子、后续掉落都会复用

3. 箱子/柜子/回复站的通用交互文本模板
   - 因为地图铺内容会很快用到

## 7. 建议的文案填写模板

你后面写文本时，建议按下面这个结构统一整理：

```md
item_id:
显示名:
类型: 武器 / 护甲 / 饰品 / 材料 / 消耗品 / 交互文本
一句话定位:
正式描述:
补充备注:
```

交互文本建议写成：

```md
对象:
PromptText:
InteractionTexts:
1.
2.
3.
GrantedItems:
- item_id x数量
```

## 8. 后续接线建议

等你把文本写好后，建议按这个顺序回填：

1. 先回填 `EquipmentCatalog.cs` 里的装备描述
2. 再回填 `SystemFeatureLabController.cs` 里的物资名称与描述
3. 最后给地图上的交互实例逐个补 `InteractionTexts` 和 `GrantedItems`

这样不会影响现有逻辑，也最容易逐步验证。
