## 1. 文档目标

这份文档专门回答一件事：

- 天赋树里到底有哪些正式节点
- 每个节点在游戏内叫什么
- 它花多少点
- 前置是什么
- 它会解锁什么牌、改什么数值、对 build 有什么意义

这次修订重点处理三件事：

- 卡牌树不再做成纯一本道
- 树结构也不再只是“主干 + 一层分叉”
- 而是改成真正的递归树状结构：任何节点都能继续长出分支，分支节点本身也能继续分叉
- 角色树把泛化节点拆成更具体的单项能力节点，并重新设计更讲究的前置顺序

## 2. 总体结构

当前推荐天赋树分为三部分：

1. `卡牌树`
- 毁灭 / `melee`
- 制衡 / `ranged`
- 创造 / `flex`

2. `角色能力树`
- 共通角色能力主干
- 从底层生存和资源能力一路往上长

3. `跨树节点`
- 用于证明 build 真的成形
- demo 阶段只开放少量

## 3. 节点设计统一规则

## 3.1 节点成本

推荐默认：

- 普通节点：`1` 点
- 高层关键节点：`1`
- 跨树关键节点：`2`

原则：

- demo 里不靠高点数制造复杂度
- 复杂度主要来自前置结构和方向取舍

## 3.2 节点层级

卡牌树统一采用：

- `入口层`
- `分化层`
- `专精层`
- `核心层`
- `招牌层`

角色树统一采用：

- `底层能力`
- `战术承载`
- `高层强化`

## 3.3 节点效果类型

每个节点的效果只允许落在以下几类：

- 解锁卡牌
- 增加分支专精值
- 提高基础动作表现
- 调整构筑预算
- 调整同名上限
- 改善学习机制
- 改善荒川能力
- 提高角色基础承载能力

## 4. 卡牌树总原则

卡牌树这次明确改成：

- 每个流派都有自己的根节点
- 根节点往外递归生长
- 任何节点都可以长出 1 到 3 个后继节点
- 某个分支节点本身也可以继续分叉
- 中后段允许部分分支汇回同一个高层节点

这样做的好处：

- 玩家不会觉得“我只是顺着一条线点下去”
- 每次成长都有更明确的打法承诺
- 同一流派玩家之间也会出现 build 差异

## 5. 毁灭分支

定位：

- 贴身爆发
- 护盾压制
- 反击
- 撞击与终结

### 结构概览

毁灭树推荐做成：

- `破门本能`
  - 向外长出 `死斗姿态`
  - 向外长出 `压舱重击`
  - 向外长出 `结构猎手`
- `死斗姿态`
  - 再向外长出 `贴身反击`
- `压舱重击`
  - 再向外长出 `盾裂处决`
- `结构猎手`
  - 再向外长出 `舰体解剖`
- 三条中层分支最后共同汇向：
  - `劫掠冲动`
  - `破舰者`
  - `舰体处决者`

## 8. 角色能力树

角色能力树这次不再保留两个过于泛化的大节点：

- `老练本能`
- `荒川增幅`

它们会被拆成更具体、可感知、可平衡的单项节点。

同时，角色树前置顺序改为更精心的梯度生长：

- 从底层能力开始
- 往上先长出单项战术承载
- 再在更高层汇合成复合型强化

也就是说，它更像“从基础体能和行动能力往上生长”，而不是一开始就点到高抽象的综合节点。

### 结构概览

底层能力：

- `应激输出`
- `生还者框架`
- `稳态电容`
- `趋前步伐`

战术承载：

- `防御校准`
- `冥想回路`
- `构筑缓存`
- `超规预留`

高层强化：

- `荒川建构增幅`
- `荒川强化协议`
- `荒川武器映射`
- `基础动作统合`

- 我要更容易打高费牌
- 我要更容易靠走位赢

它们不直接决定流派，但会强烈影响流派体验。

## 11. 与当前实现的对接建议

当前 `SystemFeatureLabController` 中的天赋节点仍是原型集。

建议接下来按这份文档推进：

1. 先替换角色树
- 因为角色树当前原型最偏工具感

2. 再替换卡牌树主干和分叉
- 让节点布局真正像树，而不是线性表

3. 最后补跨树节点
- 作为 demo 后段 build 成形的证明

## 12. 结论

这次修订后的天赋树有两个最关键的变化：

- 卡牌树从“一本道”变成“主干 + 分叉”
- 角色树从“几个大而空的综合节点”变成“从底层能力长上去的细分节点”

这样玩家在游戏内看到的将不再是：

- 顺着一条线点下去
- 点一个什么都加的大节点

而会更接近：

- 我先决定基础承载
- 再决定战术放大方向
- 最后拿到真正能代表 build 身份的核心节点

这才是更适合植入到当前游戏内的成长树方案。
