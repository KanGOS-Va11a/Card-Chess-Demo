# 打磨阶段开发主任务

时间：2026-04-23

## 1. 文档定位

本文件是当前阶段唯一的主任务拆分文档。  
后续新增开发需求默认先归入本文件，不再继续横向新增重复的“DayX清单”“补充方案”“优先级清单”。

## 2. 当前阶段的唯一目标

把项目从“通过初赛的可展示原型”推进到“一个完整区域闭合循环的高完成度可交付切片”。

当前建议采用的核心可交付切片是：

`一个完整区域 = 地图探索 -> 3-5 场战斗 -> 至少 1 场精英或 Boss -> 奖励/成长回流 -> 状态保留 -> 下一分支推进`

这条切片必须能真实验证以下系统联动：

- 地图探索
- 战斗遭遇
- 学习机制
- 构筑调整
- 装备与天赋成长
- 剧情推进
- 进度状态保留

## 3. 当前阶段的问题分组

### A. 结构性问题

- 地图交互底层仍残留非格子制临时方案
- 对话、剧情、交互、战斗入口的触发与锁输入逻辑不统一
- 构筑系统存在真源不清与刷新异常问题
- 返回地图后的状态保留机制不稳定

### B. 内容性问题

- 剧情文本仍有大量临时占位内容
- 学习机制在当前体验中的位置过低
- 非线性流程表达不足
- 玩家情绪时刻缺乏足够明确的设计呈现

### C. 表现性问题

- 背景、美术、音乐仍有临时素材残留
- 文本错字、溢出、显示不一致问题仍存在
- 天赋界面的视觉层级、布局与操作体验仍偏工具态

### D. AI 与关卡体验问题

- 敌人意图表达不够清晰
- 障碍物处理和寻路绕行不稳定
- 学习敌人能力这一核心亮点尚未被充分前置

## 4. 当前阶段的任务分层

## 4.1 P0：先修底层与体验主干

1. 地图交互改为以格子为准  
2. 对话 / 剧情 / 交互弹窗完全统一  
3. 前置 UI 弹出时锁死角色输入与角色动作状态  
4. 战斗返回地图不刷新地图物品与一次性进度  
5. 重做构筑真源，修复卡组初始化与显示不同步  
6. 重新提升学习机制在主流程中的展示位置

## 4.2 P1：完成完整区域切片

1. 把当前流程整理成可体现的非线性区域推进  
2. 丰富剧情文本，替换 AI 临时文本  
3. 补齐探索奖励、装备、学习卡、图鉴、成长的回流体验  
4. 优化敌人 AI，使其意图更明确、绕障更稳定  
5. 完成一个情绪时刻明确的关键桥段

## 4.3 P2：表现与提交打磨

1. 替换背景与音乐到正式素材  
2. 修复所有错字与文本框溢出  
3. 优化天赋界面表现  
4. 继续提升美术整体辨识度  
5. 与提案书同步收口“核心可交付切片”和“玩家情绪时刻”

## 5. 当前阶段的具体执行顺序

### 第一组：底层规则统一

1. 地图交互从球形碰撞改为格子交互  
2. 剧情触发区、交互物、出生点等统一到可绘制图层  
3. 对话弹窗、剧情文本、教程弹窗统一表现  
4. 输入锁与角色动作状态锁统一

### 第二组：状态与数据真源收口

1. 构筑系统重做真源  
2. 学习结果、图鉴、构筑、战斗初始化打通  
3. 地图返回状态与一次性交互状态打通  
4. 战斗返回场景不刷新地图物品

### 第三组：体验主干抬升

1. 调整主流程，把学习机制前置  
2. 丰富剧情文本并加入更明确的情绪时刻  
3. 把区域推进改成可感知的非线性流程  
4. 优化敌人 AI 与房间题面

### 第四组：表现收口

1. 替换背景和音乐  
2. 修正文本显示问题  
3. 天赋界面视觉重构  
4. 美术表现强化

## 6. 当前阶段的情绪目标

除了系统闭环，本阶段还必须补上玩家视角的情绪设计。

至少要明确实现一个可被提案直接描述的时刻：

玩家第一次学到某个强敌或 Boss 的独特技能，将其加入自己的卡组，并在后续战斗中用它反制同类敌人。

这个时刻的意义在于：

- 让“学习机制”从系统描述变成玩家可感知的爽点
- 让评审感受到项目不只在做规则拼接，也在思考体验记忆点

## 7. 当前阶段的约束

1. 不再新增新的主任务 md  
2. 不由助手执行 Godot 运行时测试  
3. 助手后续仅做静态审查、代码修改、构建检查与文档维护  
4. 文档归档与规范调整不写 `devlog`  
5. 只有代码功能变更才写 `devlog`
