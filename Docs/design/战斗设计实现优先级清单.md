# 战斗设计实现优先级清单

时间：2026-04-06

适用范围：
- 当前 demo 战斗系统
- 参考主文档：[序章至垣428Demo设计Wiki.md](D:/Godot/newproject/Card-Chess-Demo/docs/design/%E5%BA%8F%E7%AB%A0%E8%87%B3%E5%9E%A3428Demo%E8%AE%BE%E8%AE%A1Wiki.md)

本文目标：
- 把战斗部分从“已有接口与框架”推进到“真正可玩的策略闭环”
- 明确当前工期下，什么必须先做，什么可以延后
- 避免继续扩散式开发，优先做能直接提升 demo 完成度的内容

## 1. 当前判断

当前战斗代码已经具备较完整的框架：
- 棋盘、路径、对象注册、行动服务、卡牌运行时、敌人 AI、战斗结算、奖励边界、装备与成长边界、战斗 HUD 都已经存在

当前真正的短板不是“缺结构”，而是：
- 设计文档里已经明确的关键战斗规则，还没有全部做成真实可玩的题面
- 内容闭环存在，但强度不够，很多地方仍停留在近似表达或预留状态

因此接下来的重点应该是：
- 优先落实战斗规则与题面
- 再补内容
- 最后再补更深的系统扩展

## 2. 总优先级

### P0：必须优先完成

1. 基础题面规则做实
- 内容：
  - 可破坏障碍物
  - 不可破坏墙体
  - 缓速可通过地形/障碍
- 原因：
  - 小棋盘战斗的核心不是铺量，而是路线和站位题
  - 这部分不做实，击退、堵路、绕路、荒川能力都缺乏真正价值
- 验收标准：
  - 至少 3 类障碍/地形在战斗中有真实规则差异
  - 玩家与敌人都遵守这些规则
  - 房间布局能稳定制造路线题

2. 真正地形伤害规则接入
- 内容：
  - 电弧地形
  - 火焰地形
- 规则建议：
  - 电弧：进入、离开或停留时受伤
  - 火焰：进入或停留时受伤
- 原因：
  - 这是创造系和场景联动的核心
  - 也是 Boss 后半阶段改题面的基础
- 验收标准：
  - 地形伤害不只是视觉表现，而是真正参与战斗结算
  - 至少有 1 张卡牌和 1 个敌人会主动制造危险地形

3. 学习机制最小闭环落地
- 内容：
  - 记录敌人“上回合实际使用动作”
  - `学习` 卡读取该记录
  - 战斗胜利后按记录结果解锁普通牌或特色牌
- 原因：
  - 这是 demo 最有辨识度的机制之一
  - 比传统掉卡更符合当前世界观与玩法身份
- 验收标准：
  - 至少 1 个精英敌人、1 个 Boss 支持学习
  - 战斗结果中能稳定回传学习解锁结果

4. 普通攻击与武器边界做实
- 内容：
  - 普通攻击不再是纯硬编码固定值
  - 接入当前武器、角色成长、战斗内临时覆盖的影响
- 原因：
  - 后续 `拔枪`、武器差异、成长修正都依赖这里
  - 这是“装备影响战斗”的真正起点
- 验收标准：
  - 普通攻击至少能被武器参数影响
  - 战斗内临时换武器存在清晰入口，不需要重写攻击系统

### P1：应在 P0 后尽快完成

5. 核心卡牌按设计文档补齐
- 优先补的不是“数量最多”，而是“最能代表三个流派”的卡：
  - 毁灭：`冲撞`、`架势`、`重击`
  - 制衡：`快枪`、`震荡射击`、`戒备`
  - 创造：`电弧泄露`、`学习`、`风化`
- 验收标准：
  - 三个流派都至少有 2 到 3 张真正改变题面的卡
  - 起始牌 + 流派牌之间能形成明显打法分化

6. 精英 / Boss 战结构做实
- 内容：
  - 不是只做高数值敌人，而是做阶段与题面变化
- 推荐优先实现：
  - 1 个精英：例如偏学习机制或偏路线压迫
  - 1 个 Boss：锈船长
- 锈船长建议先做最小版：
  - 第一阶段：拉拽/压迫
  - 半血后：改地形
  - 最后阶段：召唤杂兵封路
- 验收标准：
  - Boss 战能明显体现策略上限
  - 玩家必须使用走位、卡牌与荒川能力协同

7. 荒川能力与题面的联动增强
- 当前方向：
  - 造物
  - 强化
  - 武器相关能力
- 后续重点：
  - 不只是“给数值”，而是明确改变题面
  - 例如封路、开路、制造危险区、强化某一击的突破能力
- 验收标准：
  - 荒川能力至少有 1 个用于控场、1 个用于强化、1 个用于破局

### P2：可以在 demo 主线打通后补

8. 更完整的装备联动
- 内容：
  - 战前初始化
  - 战斗内临时换武器
  - 战后回写
  - 与构筑承载、学习、超规槽位联动
- 原因：
  - 重要，但不是当前最短路径

9. 更复杂的地形扩展
- 内容：
  - 深渊杀
  - 更复杂的塌陷地格
  - 持续扩散型危险区
- 原因：
  - 可以很有趣，但容易吃工期

10. 更完整的敌人学习池
- 内容：
  - 大量敌人全部具备普通牌/特色牌双层学习奖励
- 原因：
  - 值得做，但应建立在核心 1 精英 + 1 Boss 已经可玩的前提上

## 3. 推荐实施顺序

建议按以下顺序推进：

1. 障碍物 / 缓速 / 路径规则
2. 电弧与火焰地形真实伤害
3. `学习` 机制最小闭环
4. 普通攻击受武器影响
5. 三系关键卡牌补齐
6. 精英战
7. Boss 战
8. 更完整的装备与成长联动

这样推进的好处：
- 每一步都直接提升战斗可玩性
- 每一步都能被当前 demo 场景立即验证
- 不会先掉进大系统细节，导致核心体验迟迟不成立

## 5. 当前不建议优先做的内容

以下内容不是不做，而是不应排在战斗规则闭环之前：

- 再新增更多战斗 UI 页面
- 再扩大量房间池
- 先做完整存档细节
- 一次性实现整棵天赋树所有节点
- 先补很多卡牌数量而不补卡牌的题面价值

原因：
- 这些内容会吃开发时间，但不能像“题面规则”那样立刻提升 demo 战斗质量

## 6. 最终目标

这一轮战斗完善的目标，不是让系统名词更多，而是让玩家真正体验到下面这条链路：

`路线判断 -> 障碍与地形处理 -> 卡牌出牌 -> 荒川改题面 -> 观察敌人行为 -> 学习与战后成长`

只要这条链路成立，当前 demo 的战斗部分就会真正进入“可展示作品”状态。
