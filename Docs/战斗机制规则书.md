# 战斗机制规则书

## 1. 文档定位

本文件以“规则书”方式描述棋盘战斗机制，目标是让你后续实现时有一份接近桌游或 TRPG 规则手册的规范文本。

## 2. 战斗基础定义

### 2.1 棋盘

战斗发生在一个 90 度方格棋盘上。棋盘由多个格子组成，每个格子具有：

- 坐标
- 地形
- 通行消耗
- 格子标签
- 驻留 object

### 2.2 Object

所有场上实体统一称为 `BoardObject`。

主要类别：

- 单位：玩家、敌人、召唤物
- 障碍物：掩体、箱体、电磁地板、可导电装置
- 场域：火焰区、扫描区、毒雾、电流区

### 2.3 占位规则

- 一个格子最多只能有一个单位型 object。
- 一个格子可以有多个非单位 object。
- 若某个障碍物标记为可与单位同格，则单位可以进入该格。
- 若障碍物不可与单位同格，则其所在格不可被单位占据。

## 3. 回合与时间

### 3.1 玩家回合

玩家在自己的回合中可以进行行动。玩家回合不等于只能做一件事。

推荐结构为：

- 移动阶段：可选，且每回合最多一次
- 主行动阶段：普通攻击、防御、冥想或打出卡牌

### 3.2 时间推进

时间并不是持续实时流逝，而是由玩家行动推动。

每次玩家完成一个会推进时间的行动时，系统会推进若干 `time_tick`。

时间推进会影响：

- 能量恢复
- 状态持续时间
- 投射物行为
- 场地效果结算
- 敌方计时行为

## 4. 玩家基础行动

## 4.1 移动

### 规则

- 玩家具有 `移动力`。
- 一次移动可以经过多个格子，但总消耗不能超过当前回合可用移动力。
- 每个格子的通行消耗可能不同。
- 移动本身不结束回合。
- 每回合只能进行一次移动。

### 通行消耗

建议公式：

`格子总消耗 = 地形基础消耗 + object 修正 + 状态修正`

### 路径判定

推荐流程：

1. 计算从起点到目标点的路径。
2. 按路径逐格累积消耗。
3. 若累计消耗超过移动力，则路径在上一个合法格截断。
4. 玩家移动到最终合法格。

### 移动后的状态

- `has_moved = true`
- 玩家仍可执行普通攻击、防御、冥想、打牌

## 4.2 普通攻击

### 规则

普通攻击必须满足：

- 目标在攻击距离内
- 目标在视线内
- 目标符合攻击允许对象类型

### 攻击后果

- 对目标造成伤害
- 若目标是可破坏障碍物，则其耐久降低
- 若目标生命归零，则进入破损、销毁或死亡流程
- 普通攻击后回合结束

## 4.3 防御

### 规则

防御是以结束当前回合为代价，换取防御收益的行动。

### 效果

- 回复一定护盾值
- 触发专门的减伤限制规则
- 防御后回合结束

### 护盾与减伤上限

防御不应直接等于“无脑减伤”。建议区分：

- 护盾：优先抵消伤害
- 减伤：按专门公式限制上限

示意：

1. 原始伤害进入计算
2. 按减伤公式计算可减免部分
3. 护盾吸收剩余伤害
4. 未被吸收部分才扣生命

## 4.4 冥想

### 规则

冥想代表玩家放弃进攻，转而重整手牌节奏。

### 效果流程

1. 丢弃当前手牌中的全部卡牌
2. 触发与弃牌相关的效果
3. 抽指定数量的新卡
4. 回合结束

## 4.5 打出卡牌

### 规则

- 打牌需要支付能量
- 每张牌都具有明确的目标规则与效果包
- 某些卡牌具有 `迅速` 关键词
- 某些卡牌具有 `消耗` 关键词

### 回合结束判定

- 若卡牌有 `quick`，打出后通常不结束回合
- 若卡牌没有 `quick`，则通常结束回合
- 最终以卡牌定义中的 `ends_turn` 为准

## 5. 玩家回合结束规则

玩家回合会在以下情况结束：

- 玩家完成普通攻击
- 玩家完成防御
- 玩家完成冥想
- 玩家打出一张结束回合的卡牌

玩家回合不会在以下情况自动结束：

- 单次移动完成
- 打出 `quick` 卡

## 6. 能量系统

### 6.1 基础定义

玩家有战斗能量池。

核心参数：

- `energy_current`
- `energy_max`
- `energy_regen_interval`
- `energy_regen_amount`
- `energy_regen_counter`

### 6.2 基础规则

- 进入战斗时，能量回复至当前上限
- 每经过指定数量 `time_tick`，恢复一定能量
- 能量恢复逻辑必须由时间推进系统统一处理

### 6.3 可修改项

以下项目必须允许被天赋、卡牌、遗物、剧情效果修改：

- 能量上限
- 回复间隔
- 单次回复量

## 7. 卡牌循环

### 7.1 四个牌区

战斗中的卡牌至少分为四个区域：

- 抽牌堆
- 手牌
- 弃牌堆
- 消耗牌堆

### 7.2 抽牌规则

- 抽牌时如果抽牌堆有足够牌，则直接抽取
- 抽牌堆为空时，立即将弃牌堆洗入抽牌堆
- 如果抽牌堆和弃牌堆都为空，则无法继续抽牌

### 7.3 消耗规则

- 具有 `exhaust` 的卡牌在打出后进入消耗牌堆
- 消耗牌堆中的牌并未从系统中删除
- 其他卡牌、效果或遗物可以与消耗牌堆互动

## 8. 卡牌定义

每张卡至少有以下属性：

- 类型：攻击 / 防御 / 技能
- 能量消耗
- 关键词
- 目标规则
- 基础效果
- 荒川强化效果
- 是否结束回合

### 8.1 攻击牌

主要用于：

- 单体攻击
- 直线冲撞
- 范围爆炸
- 对障碍物或环境造成破坏

### 8.2 防御牌

主要用于：

- 增加护盾
- 提供减伤
- 改变站位或创造掩体

### 8.3 技能牌

主要用于：

- 抽牌
- 回能
- 强化
- 转化地形
- 生成 object

## 9. 索敌与目标规则

### 9.1 目标获取分两步

1. 先根据几何规则找格子
2. 再根据 object 过滤规则找目标

### 9.2 视线

视线表示从攻击源到目标之间，是否存在阻挡视线的 object。

### 9.3 物理距离

物理距离是几何规则的一部分，可以是：

- 曼哈顿距离
- 切比雪夫距离
- 自定义距离规则

### 9.4 典型目标规则

#### 冲撞

- 仅允许四方向目标
- 常见为近距离
- 可附带位移或碰撞效果

#### 狙击

- 超远距离甚至全图
- 可定义是否需要视线
- 可定义是否无视障碍物

#### 导弹

- 指定一个落点格
- 对该格及周围范围造成影响
- 通常不依赖传统近战范围

## 10. 障碍物规则

### 10.1 障碍物不等于墙

障碍物不一定不可通行。

障碍物可能具备以下任意组合：

- 可通行
- 不可通行
- 挡视线
- 不挡视线
- 可受伤
- 不可受伤
- 可与单位同格
- 不可与单位同格

### 10.2 可破坏障碍物

可破坏障碍物至少要支持：

- 最大耐久
- 当前耐久
- 受伤反馈
- 破损状态
- 销毁结果

### 10.3 典型例子

#### 电磁地板

- 属于障碍物或特殊场地 object
- 可通行
- 可与单位同格
- 可在进入或时间推进时触发额外效果

#### 掩体箱

- 不可通行
- 可能挡视线
- 可被破坏
- 销毁后格子恢复可通行

## 11. 荒川的帮助

### 11.1 荒川的战斗定位

荒川不是常规场上单位，而是：

- 地形改造者
- 卡牌强化者
- 特殊 object 操作者

### 11.2 荒川强化卡牌

每张可被强化的卡建议至少有两套效果：

- 基础效果
- 被荒川强化后的效果

### 11.3 荒川与 object 的互动

荒川可以：

- 生成新的障碍物
- 转换已有障碍物
- 让普通地块变为可导电、可爆炸或危险地块
- 将障碍物转为有利或不利 object

## 12. 时间推进中会发生的事情

每次时间推进，系统应按统一顺序更新。推荐顺序：

1. 更新能量回复计数
2. 更新持续状态
3. 更新场地效果
4. 更新投射物
5. 更新 object 的持续行为
6. 更新敌方待执行行为计数

## 13. 一回合示例

### 示例 1：移动后普通攻击

1. 玩家开始回合
2. 玩家移动 3 格，消耗对应移动力，回合未结束
3. 玩家选择普通攻击
4. 系统检查距离与视线
5. 攻击命中目标
6. 回合结束
7. 时间推进

### 示例 2：迅速卡连打

1. 玩家开始回合
2. 玩家打出一张 `quick` 卡
3. 系统结算卡牌效果，但不结束回合
4. 玩家可继续打牌或执行其他允许的主行动
5. 当玩家执行结束回合的行动时，回合才结束

### 示例 3：冥想

1. 玩家开始回合
2. 玩家不移动也不攻击，选择冥想
3. 手牌全部进入弃牌堆
4. 抽指定数量新卡
5. 回合结束
6. 时间推进

## 14. 规则实现优先级

### 第一优先级

- BoardObject 模型
- 占位规则
- 房间初始化
- 移动系统
- 普通攻击系统
- 能量系统
- 抽牌 / 弃牌 / 洗牌 / 消耗牌堆

### 第二优先级

- 荒川强化
- 可破坏障碍物
- 更复杂 targeting
- 时间推进中的投射物与场地效果

## 15. 结语

你这套战斗机制最难的部分不是“做出一张卡”，而是先把以下底层规则统一：

- object 如何存在于格子上
- 单位与障碍如何同格或互斥
- 行动何时结束回合
- 时间何时推进
- 卡牌如何统一索敌

只要这些规则先写清楚，后面的卡牌设计、敌人设计、荒川能力设计都会变得非常稳。
