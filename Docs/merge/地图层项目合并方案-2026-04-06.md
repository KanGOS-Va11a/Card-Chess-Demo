## 1. 背景

本次合并的对象是：
- 主项目：`D:\Godot\newproject\Card-Chess-Demo`
- donor 地图工程：原位于 `Merge/Card-Chess-Demo`

由于双方曾从较早版本分线开发，地图、战斗、全局状态、场景目录和脚本路径都出现了明显分叉，因此当时采用的不是整工程覆盖或普通 `git merge`，而是：

`以当前主项目为主干 -> donor 作为地图层内容提供方 -> 分批迁入资源与场景 -> 核心脚本手工并轨 -> 保住 battle 真源`

## 2. 当时的合并原则

### 2.1 以谁为真源

- battle 真源：当前主项目
  - 以 `GlobalGameSession`
  - `BattleRequest / BattleResult`
  - battle 场景与卡牌/成长/构筑接口为准

- 地图内容真源：donor 工程
  - 以 donor 的 `Scene01~Scene06`
  - 地图演出、流程、走格结构、视觉控制器为准

### 2.2 明确不做的事

- 不对 donor 工程整目录覆盖
- 不引入第二套 battle 全局单例
- 不直接混用 donor 的 `Core/BattleRequest.cs` / `Core/BattleResult.cs`
- 不整目录覆盖 `Scripts/Map`
