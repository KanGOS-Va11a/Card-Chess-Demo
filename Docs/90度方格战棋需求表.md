# 90度方格战棋需求表

## 1. 目标摘要

- 棋盘中的玩家、敌人、障碍物统一按 `BoardObject` 管理。
- 障碍物支持受伤、破损、摧毁、转化、生成。
- 一部分障碍物允许与单位同格。
- 玩家与敌人作为单位对象，不能共享同一格单位槽。
- 房间布局通过预设方案初始化 board。
- 战斗规则支持“一回合内先移动，再进行主行动”。

## 2. 核心需求总表

| ID | 模块 | 需求 | 优先级 | 验收标准 |
| --- | --- | --- | --- | --- |
| R-01 | Board Object | 玩家、敌人、障碍物统一为 `BoardObject` | P0 | 三者都能在 registry 中以统一对象管理 |
| R-02 | Cell Occupancy | 格子支持单位槽和可共存对象集合 | P0 | 玩家与敌人不能同格，单位可与可通行障碍同格 |
| R-03 | 障碍物生命 | 障碍物支持 HP、受伤、破损、销毁 | P0 | 至少一种障碍物可被普通攻击或卡牌破坏 |
| R-04 | 可通行障碍物 | 支持不阻挡移动但仍为 object 的障碍物 | P0 | 如电磁地板可与单位同格并触发效果 |
| R-05 | Board 初始化 | 棋盘可根据房间布局方案初始化 | P0 | 输入房间布局后可稳定生成地形、障碍、出生点 |
| R-06 | 单回合移动 | 玩家回合内只能移动一次，但移动不结束回合 | P0 | 移动后仍能攻击、防御、冥想或打牌 |
| R-07 | 普通攻击 | 支持攻击范围、视线、距离联合判定 | P0 | 只有在视线内且距离合法时才可攻击 |
| R-08 | 防御 | 支持护盾回复与减伤上限公式 | P0 | 防御后减伤表现符合专用公式 |
| R-09 | 冥想 | 弃掉所有手牌并抽指定数量 | P0 | 冥想后手牌重置且回合结束 |
| R-10 | 卡牌出牌 | 支持迅速与非迅速卡的回合结束差异 | P0 | `quick` 卡不结束回合，其他卡通常结束回合 |
| R-11 | 能量系统 | 支持能量上限、当前值、回复间隔、回复速度 | P0 | 每经过指定 time tick 回复能量 |
| R-12 | 能量解耦 | 天赋或卡牌可独立修改能量上限和回复速度 | P0 | 不需要修改出牌主流程即可改变回复规则 |
| R-13 | 卡牌循环 | 支持抽牌堆、手牌、弃牌堆、消耗牌堆 | P0 | 抽牌堆空时立即洗弃牌堆 |
| R-14 | 消耗关键词 | `exhaust` 卡进入消耗牌堆，不从系统中真删除 | P0 | 其他卡可与已消耗卡牌互动 |
| R-15 | 目标规则 | 卡牌可支持冲撞、狙击、导弹等不同索敌模式 | P0 | 至少 3 类不同目标规则能共存 |
| R-16 | 格子索引管理 | 统一 `cell -> objects` 与 `object_id -> object` 管理 | P0 | 卡牌、普通攻击、荒川能力统一复用 |
| R-17 | 荒川强化 | 卡牌定义支持基础效果和荒川强化效果 | P0 | 至少一张卡可在被强化后改变效果包 |
| R-18 | 荒川造物/转化 | 荒川可生成或转换障碍物 object | P0 | 至少一种能力可生成或转换障碍 |
| R-19 | 时间推进 | 玩家的行动定义时间推进 | P0 | 时间推进会更新能量回复、状态、投射物 |
| R-20 | 房间兼容性 | 战斗系统可接受地图层传入的房间/遭遇请求 | P0 | 可从外部布局方案初始化战斗 |

## 3. Board 需求

### 3.1 Cell 结构需求

每个格子至少要能表达：

- 地形
- 单位占位槽
- 可共存对象集合
- 阻挡对象集合
- 地块标签
- 通行消耗

### 3.2 Object 标签需求

建议最少支持这些标签：

- `destructible`
- `passable`
- `blocks_movement`
- `blocks_los`
- `conductive`
- `hazard`
- `stackable_with_unit`

## 4. 玩家行动需求

### 4.1 移动

- 使用移动力系统
- 每格消耗可变
- 每回合最多一次移动
- 移动不结束回合

### 4.2 普通攻击

- 攻击后回合结束
- 攻击范围判定与视线判定分离
- 后续卡牌应能复用普通攻击的判定组件

### 4.3 防御

- 增加护盾值
- 使用专门减伤公式
- 攻击后回合结束

### 4.4 冥想

- 丢弃全部手牌
- 抽固定张数
- 回合结束

### 4.5 打出卡牌

- 能量足够才可打出
- `quick` 卡打出后回合不结束
- 非 `quick` 卡打出后回合结束

## 5. 卡牌系统需求

### 5.1 卡牌定义最少字段

- `card_id`
- `card_type`
- `cost_energy`
- `keywords`
- `target_rule_id`
- `effect_bundle_id`
- `arakawa_effect_bundle_id`
- `ends_turn`

### 5.2 卡牌类型

- `attack`
- `defense`
- `skill`

### 5.3 关键词

- `quick`
- `exhaust`
- 后续可扩：`retain`, `ethereal`, `pierce`, `homing`

## 6. 能量系统需求

### 6.1 字段

- `energy_max`
- `energy_current`
- `energy_regen_interval`
- `energy_regen_amount`
- `energy_regen_counter`

### 6.2 行为

- 进入战斗时回复至上限
- 每经过指定时间回复
- 可被卡牌、天赋或 relic 修改上限和回复速度

## 7. 房间初始化需求

### 7.1 输入

- 房间布局方案
- 战斗请求
- 随机种子

### 7.2 输出

- 初始化完成的 BoardState
- object 注册表
- 玩家出生点与敌方出生点

## 8. 实现优先级

### P0 先完成

- BoardObject 模型
- Cell 占位规则
- Board 初始化
- 移动、普通攻击、防御、冥想
- 能量系统
- 基础卡牌循环
- 统一索敌服务

### P1 再补

- 可破坏障碍的多阶段破损
- 更复杂的 object 同格交互
- 多种荒川转化效果
- 更复杂的卡牌关键词

## 9. 验收用例建议

- 玩家可站在电磁地板上，但不能站进敌人所在格。
- 玩家可攻击可破坏箱体并将其销毁。
- 玩家本回合移动后仍可普通攻击。
- 玩家打出迅速卡后仍可继续打牌或防御。
- 抽牌堆为空时会自动洗弃牌堆。
- 荒川强化后的卡牌会调用另一套效果。
