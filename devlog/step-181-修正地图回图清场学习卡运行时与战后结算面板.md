# step-181 修正地图回图清场、学习卡运行时与战后结算面板

时间：2026-04-08

本次目标：
- 修正战斗胜利后地图敌人和教程剧情没有消失的问题
- 把 `学习` 卡正式接入运行时逻辑
- 补最小战后结算面板
- 修正 `C` 菜单中经验显示不一致的问题

本次完成内容：

1. 地图回图清场
- `MapResumeContext` 新增：
  - `SourceInteractablePath`
  - `EncounterId`
- 进入战斗时会记录“由哪个地图交互物发起”
- 战斗胜利后会把该交互物在返回地图快照中直接标记为禁用
- 同时记录：
  - `ClearedEncounters`
  - `UsedInteractables`

2. Scene01 教程敌人与剧情不再重复
- `Scene01TutorialController` 现在会检查教程遭遇是否已经清除
- 若已清除：
  - 直接移除地图敌人
  - 不再重复进入敌人相关教程剧情

3. 学习卡正式加入游戏
- 新增 `card_learning`
- 当前逻辑：
  - 对敌方单位使用
  - 若目标处于“特殊可学习状态”，记录特色奖励牌
  - 否则记录普通奖励牌
- `Scene01` 当前敌人的特殊可学习状态定义为：
  - 生命值低于一半

4. 学习奖励改为按运行时记录发放
- 胜利时不再固定双发两张奖励牌
- 改为读取战斗结果 runtime flags 中的 `learned_card_ids`
- 当前 `Scene01` 教程敌人的两档学习奖励：
  - 普通：`card_patrol_strike`
  - 特殊：`card_last_chase`

5. 战后结算面板
- 战斗胜利返回地图前会先弹出一个最小结算面板
- 当前显示：
  - 经验
  - 解锁卡牌
- 继续方式：
  - `E`
  - `Enter`

6. 经验显示一致性修正
- debug 模式下把等级抬到 `Lv.3` 时
- 现在会同步把经验值至少抬到该等级的经验底线
- 避免出现：
  - 当前经验值
  - 下一级所需经验
  - 玩家当前等级
  三者互相不一致

当前建议测试：
1. 进入 `Scene01`
2. 击败教程敌人
3. 确认：
   - 先出现结算面板
   - 返回地图后敌人消失
   - 敌人相关剧情不再重播
4. 使用 `C` 查看：
   - 经验显示是否一致
   - `学习` 卡是否已解锁
   - 学到的奖励牌是否在图鉴 / 构筑中出现
