# step-185 整理适合游戏内植入的天赋树正式方案文档

时间：2026-04-08

本次目标：
- 基于设计 wiki 与现有实现，整理一份真正适合植入游戏内的天赋树方案
- 不是继续扩抽象设定，而是把入口、结构、交互和成长节奏写成正式方案

本次新增文档：
- `Docs/design/游戏内天赋树植入方案-2026-04-08.md`

文档核心结论：
1. 继续保留双树
- 卡牌树
- 角色能力树

2. 必须共用资源
- 统一使用专精点
- 让玩家在“构筑成长”和“角色成长”之间真正取舍

3. 必须双树同屏
- 不建议拆成两个 panel 页面
- 更适合做成同一张可拖拽、可缩放的成长地图

4. 节点要少而关键
- 卡牌树每系 4 个主节点
- 角色树 6~8 个节点
- 总量控制在 18~22

5. 天赋树应是 build 中枢
- 与构筑
- 装备
- 学习机制
- 战斗奖励
  共用同一条循环

文档用途：
- 作为后续从 `SystemFeatureLabController` 原型节点收敛到正式节点集的设计依据
- 为后续 UI 和数值整理提供统一目标
