# step-177 输出三天战斗内容冲刺顺序表

时间：2026-04-07

本次目标：
- 针对当前项目“3 天内完成战斗相关核心内容”的要求，输出一份可直接执行的短期排期文档
- 明确哪些内容应优先完成，哪些内容在这 3 天内只保留最小骨架

本次新增文档：
- `Docs/design/三天战斗内容实现顺序表-2026-04-07.md`

文档核心结论：
1. 三天内不应四线并行平均推进
- 不建议同时平均投入“战斗内容实现 / 战后结算深化 / 音频系统 / 存档系统”
- 应改为“先战斗内容，后结算闭环，再最小音频，存档不深化”

2. Day 1 聚焦战斗内容本体
- 优先落地关键卡牌、敌人差异、装备效果、普通攻击与武器边界
- 目标是做出真正可玩的内容包，而不是继续堆接口

3. Day 2 聚焦战后结算与成长
- 落地 `学习` 机制最小闭环
- 接实胜利 / 战败 / 逃跑三种结果
- 确保 `GlobalGameSession` 能正确回写成长和奖励状态

4. Day 3 聚焦最小音频系统与整体验收
- 先做地图 / 战斗 BGM 与关键 SFX 的基础接线
- 再完成整条展示链的稳定性检查与高影响 bug 收尾

本次产出作用：
- 给接下来 3 天的实现顺序定下明确边界
- 避免在短工期里把时间分散到低回报的大系统深化上
- 为后续真正开始做 Day 1 的卡牌 / 敌人 / 装备落地提供执行顺序
