# step-156 把冲撞升级为真实逻辑击退版

时间：2026-04-06

背景：
- 上一版 `冲撞` 已经可用，但仍属于“最小可玩版”：
  - 主角能冲到目标面前
  - 能造成基础伤害
  - 只能用“目标背后是否受阻”近似处理追加伤害
- 用户新的要求是：
  - 敌人在直线距离上可作为目标
  - 主角移动到敌人相邻位置
  - 敌人被真实击退
  - 若击退路径内撞到障碍物，则停下
  - 敌人和障碍物都受到伤害
  - 这里的“障碍物”也包括其他敌人

本次实际改动：

1. `冲撞` 改为真实逻辑击退
- 文件：
  - `Scripts/Battle/BattleSceneController.cs`
- 内容：
  - `TryApplyRamCard(...)` 已重写
  - 当前行为为：
    - 选中直线 `2` 格内目标
    - 主角尝试移动到目标面前一格
    - 目标先吃基础 `3` 点伤害
    - 若未死亡，则沿着攻击方向尝试被击退，最多 `2` 格
    - 每一步都会检查：
      - 是否越界
      - 前方是否已有单位或障碍物
      - 前方格是否可占据
    - 若路径被阻挡：
      - 目标停在碰撞前最后一个可站立格
      - 目标额外受到 `4` 点撞击伤害
      - 被撞对象也额外受到 `4` 点伤害

2. “障碍物包括其他敌人” 已落实
- 当前击退时判定阻挡对象的范围包括：
  - 单位
  - 障碍物
  - 任何 `BlocksMovement = true` 或 `!StackableWithUnit` 的对象
- 因此其他敌人现在也会作为冲撞的碰撞对象参与结算

3. 击退后的停位已真正写入棋盘逻辑
- 目标不再只是吃一个带方向的伤害数值
- 当前会真实改动棋盘上的目标格位置
- 后续其它卡或敌人技能如果也要用击退，这套逻辑可以继续抽出来复用

4. 测试入口保持不变
- `冲撞` 仍然会随 debug 测试链被注入到开局手牌
- 当前开局仍能直接测试：
  - `处决`
  - `拔枪`
  - `电弧泄露`
  - `冲撞`

当前结果：
- `冲撞` 已经从“文档效果近似版”升级成“真实逻辑击退版”
- 现在更接近设计 wiki 中“冲到目标面前 -> 目标被击退 -> 撞到东西双方受伤”的实际效果

验证：
- 执行 `dotnet build`
- 结果：编译通过，0 error

说明：
- 当前实现仍优先保证规则可玩，不追求复杂击飞特效
- 目标位置变化现在是逻辑真实的
- 如果后续还要把“击退路径动画”也做成完整的逐格推飞表现，可以在这个逻辑基础上再补表现层动画
